## Description
Easily deploy Helm charts to your managed Kubernetes cluster directly from the cloud UI.
Specify the repository, chart version, and values overrides — the platform will generate and apply everything for you.
No need to configure access or run commands manually: just provide the chart details and get your application running.

## Short description
Install any Helm chart in your Kubernetes cluster without using kubectl or the Helm CLI.

## Tutorial
Provide the chart repository, chart name, version, and any custom values overrides

## Usage
When you deploy a chart, all its resources will be created under the release `<release-name>-child` in the namespace you choose during installation.

You can easily check the created resources using the [Kubernetes Dashboard](https://console.eu.nebius.com/applications/overview/nebius/kubernetes-dashboard).

## Use cases

- Quickly deploy your own Helm charts.
- Test applications or services in a Kubernetes namespace.
- Customize deployments with your own values and configurations.
